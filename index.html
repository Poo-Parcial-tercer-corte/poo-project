Introducción

Introducción a la POO

La Programación Orientada a Objetos es un paradigma utilizado para organizar programas mediante clases y objetos.

¿Qué es la POO?

La Programación Orientada a Objetos permite crear software organizado utilizando objetos que representan elementos del mundo real. Este paradigma facilita el mantenimiento, reutilización y escalabilidad del código.

Organización

El código se estructura mejor.

Reutilización

Polimorfismo y Sobreescritura

Capacidad de un objeto de tomar múltiples formas mediante métodos que se comportan distinto en subclases.

Ejemplo_Polimorfismo.java
// Ejemplo 1: Sobreescritura (@Override)
class Animal { void hablar() { System.out.println("Sonido..."); } }
class Perro extends Animal { @Override void hablar() { System.out.println("Guau!"); } }

// Ejemplo 2: Polimorfismo Dinámico
Animal miMascota = new Perro();
miMascota.hablar(); // Imprime: Guau!

Ejercicio 1

Crea una clase 'Figura' con un método area() y sobreescríbelo en 'Circulo' y 'Cuadrado'.

Ejercicio 2

Usa una lista de tipo 'Animal' para almacenar gatos y perros, y llama al método hablar() en un ciclo.

Clases Abstractas e Interfaces

Define contratos que las clases deben seguir para asegurar una estructura sólida.

Interfaces.java
// Ejemplo 1: Interfaz
interface Volador { void volar(); }
class Avion implements Volador { public void volar() { /* código */ } }

// Ejemplo 2: Clase Abstracta
abstract class Empleado { abstract double calcularSueldo(); }

Manejo de Errores

Control de fallos en tiempo de ejecución para evitar que la aplicación se detenga.

TryCatch.java
// Ejemplo 1: Bloque Try-Catch
try {
    int data = 50 / 0;
} catch (ArithmeticException e) {
    System.out.println("No se puede dividir por cero");
}

// Ejemplo 2: Uso de finally
finally { System.out.println("Cierre de recursos siempre ejecutado"); }
